记jarbas渗透过程的问题

Jarbas1.0是一个难度为初级的Boot2root/CTF挑战

描述：向90年代末的巴西搜索引擎致敬。

实验目的：获取目标root shell；获取flag

实验环境：vmware，kali操作，jarbas作靶机
内网桥接

https://xz.aliyun.com/t/3092

本篇只对遇到的问题做记录，不对整个渗透过程作具体阐述

端口扫描

工具：nmap

实验指导给出的命令为

nmap -A [IP]

也可以使用

nmap -sV -p- [IP]

个人感觉会更加便利

目录爆破

工具：kali自带的dirb（类似于windows平台的御剑一类的工具）

dirb的默认字典是/user hare/dirb/wordlist/common.txt

合天提供的实验环境可能是采用了做好的强力字典，故实验指导书中能直接爆破。而kali自带的字典是爆破不出来的，但网上提供了另一种方式：
用扩展名探测，如php,html；

dirb http://xxx... :[端口] -x .php(.html)   #dirb的-x参数，代表匹配扩展名，会将自店内的内容加入扩展名进行爆破

如此就能爆破出字典中原不存在的access.html（原字典中含有access）
当然如果你有一个像大表哥那样的大字典也就没那么多事儿了

LinkList

发表于 2019-09-15 更新于 2019-10-08

便于使用指针的访问形式的结构体命名方式

在完成数据结构第一次实验报告的时候，被要求完成单链表的交并差运算。

结构体和主函数都是由老师写好的，没有注意结构体的命名形式。看着满屏的LinkList，故以为LinkList为结构体的别名。

在声明变量和申请内存时就使用了如下方式

LinkList *r = La;
a = (Linklist *)malloc(sizeof(Linklist));

编译时会报错

cpp|46|error: request for member 'data' in '* r', which is of pointer type 'LinkList {aka LNode*}' (maybe you meant to use '->' ?)|

于是回头查看结构体定义部分

typedef struct LNode
{
    ......
}LNode,*LinkList;

发现了如上的的定义形式，即 *LinkList

如此定义时 LinkList相当于 Lnode *

遂将源代码改为

**LinkList** r = La;
a = (**Linklist** )malloc(sizeof(**Lnode**));

这种定义方式虽然方便了程序员定义指针，但是更容易混淆和导致误解

有利有弊吧，项目小组内部商量好了的话感觉还是挺好使的
